# External-Beta Feedback Intake

Record one report per rehearsal:

- manifest digest and repository heads;
- operating system and architecture;
- attempted component and command;
- expected and observed result;
- reproducible steps;
- logs with secrets removed by the operator;
- severity: blocking, degraded, confusing, or suggestion;
- rollback result;
- whether any safety boundary was approached.

Feedback is planning input. It does not authorize a code change, release, or
promotion.
