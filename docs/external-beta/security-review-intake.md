# External-Beta Security Review Intake

Reviewers should examine approval digest binding, policy evidence integrity,
transactional mutation and rollback, sandbox boundaries, observer
authentication, evidence replay, and denial of authority substitution.

Each finding must identify the tested commit, source path, reachable trust
boundary, preconditions, impact, reproduction method, and recommended gate.
Do not include credentials, tokens, private keys, or live provider traffic.

The review is complete only when high and critical findings have an owner and
the beta launch decision remains separate from this intake.
